yare.io is a real-time 2-player (1v1) strategy game where you control your units (cats) via JavaScript. You win by destroying the opponent’s base.

Yare.io is still in beta. Please share any bugs you find on our Discord server

Game board

Yare.io is played on a board sized 4400 × 2700 units. It’s an Euclidian plane starting [0, 0] at the top left and [4400, 2700] at the bottom right. Barricade and player base positions shown below.

Cat

Cats are the game’s units. They are JavaScript objects with various properties and methods listed below.

2 ways to access the cat object

cats[id]

object

The cats object contains all of the game’s cats. E.g. cats[‘jane2’] returns the cat with id = ‘jane2’

my_cats[index]

array

my_cats array contains only your cats. E.g. my_cats[0] is your oldest cat.

You can also access your own cats by their id. E.g. jane2.position, but that’s highly impractical in the real game outside of the tutorial.

Properties

Methods

Method Argument Can be used by
pew(target) target is an object Circles Squares Triangles

Transfers 1 energy unit from itself into target. Max distance of the energy transfer is 200 units.

If target is an enemy cat or a base, the target takes damage of 2 energy

myCats[0].pew(my_cats[1]); 
var enemy = cats['jane7']; 
my_cats[0].pew(enemy);
my_cats[1].pew(enemy);

If a cat or base has negative energy at the end of a tick, it will die.

move(target) target is an [x, y] array Circles Squares Triangles

Moves cat to target with a speed of 20 units per tick

my_cats[0].move([800, 900]);
merge(target) target is a friendly cat Circles Squares Triangles

Merges a cat into another friendly cat. Target needs to be within 10 units radius.

for (i = 0; i < 5; i++){ 
  my_cats[i].move(my_cats[6].position);
  my_cats[i].merge(my_cats[6]);
}
divide() no argument Circles Squares Triangles

Divides cat back into all the cats that were merged into it. Energy is redistributed evenly.

my_cats[9].divide()
jump(target) target is an [x, y] array Circles Squares Triangles

Teleports a cat into a new location up to 300 units away. Costs half of cat's energy_capacity. Target can not be within 100 unit distance from a barricade. If target is not a valid location, the jump will land in the nearest valid location from target

explode() no argument Circles Squares Triangles

Cat explodes, killing itself and dealing 10 damage to all enemy cats within 100 radius.

shout(message) max. 20 char string Circles Squares Triangles

Shows a message above the cat as a light-weight in-game communication. Useful for some debugging as well.

set_mark(label) label is a string (max. 60 char) Circles Squares Triangles

Updates the mark property of the cat. That can be useful for e.g. marking which cat is doing what

All instances of a method are executed together for both players every tick on the server in the same order as the table above.

First, all pew() methods are calculated, then all movements, then all merge() and so on. Except for explode() that happens together with pew().

E.g. if two cats with the same amount of energy are attacking each other, they will die at the same moment.

Base

Base is the birth-place for new cats, automatically generating new cats when it has enough energy. If there is an enemy in its sight, the base stops producing new cats, keeping all energy to defend itself.

Energy needed to generate a new cat (happens automatically once per tick)

25-400 360–800 90–300 ?

The base won’t generate new cats if there is an enemy in its sight (400 radius). It will keep all incoming energy from friendly cats (until it reaches its energy_capacity) to defend itself.

Energy

Energy is the only resource in the game. Cats use it to create new cats, transfer energy between themselves and attack with it.

Barricade properties

Property Type Value
id string “barricade_a“, “barricade_b” and "barricade_c" game board
structure_type string "barricade"
position array [x, y] coordinates (see game board)
collision_radius number Radius for collision detection

All “movement” of energy in the game is done via the pew(target) method. See the Cat methods section.
